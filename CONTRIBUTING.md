# Contributing to Cinema Platform Docs

Thank you for investing your time in contributing to our project!

> _(For the Ukrainian version of this guide, please read [CONTRIBUTING_ua.md](./CONTRIBUTING_ua.md))_

## Code of Conduct

Keep it minimal. We value clean code, clear naming, and straightforward architecture over heavy comments.

## How to Contribute

1. **Fork & Clone:** Fork the repository and clone it locally.
2. **Branching:** Create a branch for your feature (`feat/custom-header`) or fix (`fix/api-typo`).
3. **Content Guidelines:** - Write concisely.
   - Use Markdown (`.md` or `.mdx`) for all articles in `src/content/docs/`.
   - Ensure the frontmatter (like `title`) is present.
4. **Formatting:** Run `bun run format` before committing to ensure Prettier + Tailwind standards are met.
5. **Pull Requests:** Keep PRs focused. Describe _what_ changed and _why_.
